# WayMaker

> 교회의 복잡한 A/V 방송·네트워크 장비 결선도와 라우팅 정보를, **클릭·드래그·키보드만으로** 그리고 저장·공유·인쇄하는 웹 도구.

완성된 배선도 한 장을 WayMaker에서는 **WayMap(웨이맵)** 이라고 부릅니다.

> 🚧 **개발 중입니다.** 기획한 6대 핵심 기능이 모두 동작하며, 장치 라이브러리 확장과 세부 다듬기를 진행하고 있습니다.

---

## 핵심 철학

> **초보자는 단순하게 필요한 만큼 쉽게. 숙련자는 자신이 원하는 모든 정보를 표시할 수 있도록.**

- 처음 온 봉사자는 "마이크 → 믹서 → 스피커"를 선으로 잇기만 하면 됩니다.
- 음향감독은 같은 배선도에 단자 종류·라벨·케이블 길이까지 정밀하게 기록할 수 있습니다.

## 주요 기능

- **장치 추가 (드래그 앤 드롭)** — 팔레트에는 장치 *종류*(디지털/아날로그 믹서, 스테이지박스, 유선/무선 마이크, DI, 건반, 스피커류, 비디오 스위처, 카메라, 프로젝터, 디스플레이, XLR 랜 익스텐더 등)만 있고, 캔버스에 떨어뜨리면 **제조사 → 모델**(예: Behringer X32 Compact)을 고르거나 **기본 장치(Default)** 로 추가합니다.
- **장치 스펙·I/O 상세 설정** — 포트마다 이름·방향(입력/출력/양방향)·단자 종류·좌우 표시·신호 종류를 편집하고, 장치에 **커스텀 이미지**를 등록할 수 있습니다.
- **포트 단위 정밀 결선** — "A 장비 ↔ B 장비"가 아니라 **"A의 1번 포트 ↔ B의 2번 포트"** 수준으로. 케이블 양 끝의 **단자(암/수 등)·라벨·길이·메모**도 기록합니다.
- **장소(구획) 배치** — 방송실·앰프실·무대·강대상·청중석을 색상 박스로 배치하고 크기를 조절, 그 위에 장비를 올립니다.
- **신호 흐름 추적** — 마이크나 악기를 클릭하면 그 신호가 실제로 거치는 경로가 애니메이션으로 표시됩니다. 장치 내부 라우팅(예: XLR 랜 익스텐더는 채널 1 → 반대편 채널 1로만)까지 반영해 **정확한 채널 흐름**을 보여줍니다.
- **저장·공유·출력** — 범용 **JSON 파일**(다시 불러오기), **PNG**, **A4/A3 PDF**, 인터넷 없이 보는 **자기완결형 인터랙티브 HTML**, **HTML 클립보드 복사**. 작업물은 **자동저장**됩니다.

## 개인정보 보호

WayMaker는 **어떤 개인정보도 수집하지 않습니다.**

- 모든 데이터 처리는 **사용자의 브라우저(로컬)** 에서만 이뤄집니다.
- 서버로 전송하는 데이터가 없으며, 외부 분석·추적 스크립트를 사용하지 않습니다.
- 작성한 WayMap은 사용자가 직접 내려받는 파일과 브라우저 로컬 저장소에만 존재합니다.

## 기술 스택

- [Next.js](https://nextjs.org) (App Router) + TypeScript — 정적 빌드(백엔드 없음)
- [React Flow (`@xyflow/react`)](https://reactflow.dev) — 노드·포트·연결 다이어그램 엔진
- [Zustand](https://github.com/pmndrs/zustand) — 상태 관리
- [Tailwind CSS](https://tailwindcss.com)
- [html-to-image](https://github.com/bubkoo/html-to-image) · [jsPDF](https://github.com/parallax/jsPDF) — 이미지/PDF 내보내기

백엔드가 없는 클라이언트 중심 구조라 정적 호스팅(예: Vercel)에 그대로 배포할 수 있습니다.

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # 프로덕션 빌드
npm run lint     # 린트
```

## 장치 라이브러리에 기여하기

내장 장치는 **데이터**로 관리되어 추가가 쉽습니다 — [`src/lib/devices.ts`](src/lib/devices.ts):

- **종류(`DEVICE_TYPES`)** — 팔레트에 보이는 단위. 라벨·카테고리·기본 포트(Default)를 가집니다.
- **모델(`DEVICE_MODELS`)** — 한 종류에 속한 제조사/모델. 실제 포트 스펙을 가집니다.

새 제조사/모델은 `DEVICE_MODELS` 배열에 객체 하나만 추가하면 됩니다. 직접 추가해 쓰거나, **Pull Request**로 장치 목록을 함께 늘려 주세요.
