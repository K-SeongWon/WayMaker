# WayMaker

> 교회의 복잡한 A/V 방송·네트워크 장비 결선도와 라우팅 정보를, **클릭·드래그·키보드만으로** 그리고 저장·공유·인쇄하는 웹 도구.

완성된 배선도 한 장을 WayMaker에서는 **WayMap(웨이맵)** 이라고 부릅니다.

> ⚠️ **개발 초기 단계입니다.** 현재는 핵심 편집 구조를 검증하는 프로토타입(M1) 수준이며, 아래 기능 중 일부만 동작합니다. 로드맵을 참고하세요.

---

## 핵심 철학

> **초보자는 단순하게 필요한 만큼 쉽게. 숙련자는 자신이 원하는 모든 정보를 표시할 수 있도록.**

- 처음 온 봉사자는 "마이크 → 믹서 → 스피커"를 선으로 잇기만 하면 됩니다.
- 음향감독은 같은 배선도에 단자 종류·라벨·케이블 스펙까지 정밀하게 기록할 수 있습니다.

## 주요 기능 (목표)

- **장비 드래그 앤 드롭** — 오디오 믹서(Behringer X32 등), PC, 스피커, 무선마이크, 다이렉트박스, 디지털 스테이지박스, 네트워크 장비 등을 끌어다 배치
- **장비 스펙·I/O 상세 설정** — 모델이 정해진 장비는 포트 자동, 그 외에는 입력 단자 종류·개수·전원 타입 등을 직접 정의
- **포트 단위 정밀 결선** — "A 장비 ↔ B 장비"가 아니라 **"A의 1번 포트 ↔ B의 2번 포트"** 수준으로
- **장소 구조 배치** — 방송실·앰프실·무대·강대상·청중석을 박스로, 원하면 벽·문·치수까지
- **저장·공유·출력** — 범용 **JSON 파일** / 오프라인 인터랙티브 **HTML** / HTML 클립보드 복사 / A4·A3 **PDF**
- **신호 흐름 애니메이션** — 마이크나 악기를 클릭하면 그 신호가 거치는 경로를 따라 흐름을 시각화

## 개인정보 보호

WayMaker는 **어떤 개인정보도 수집하지 않습니다.**

- 모든 데이터 처리는 **사용자의 브라우저(로컬)** 에서만 이뤄집니다.
- 서버로 전송하는 데이터가 없으며, 외부 분석·추적 스크립트를 사용하지 않습니다.
- 작성한 WayMap은 사용자가 직접 내려받는 파일로만 보관됩니다.

## 기술 스택

- [Next.js](https://nextjs.org) (App Router) + TypeScript
- [React Flow (`@xyflow/react`)](https://reactflow.dev) — 노드·포트·연결 다이어그램 엔진
- [Zustand](https://github.com/pmndrs/zustand) — 상태 관리
- [Tailwind CSS](https://tailwindcss.com)

백엔드가 없는 클라이언트 중심 구조라 정적 호스팅(예: Vercel)에 그대로 배포할 수 있습니다.

## 개발 시작하기

```bash
npm install
npm run dev
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 을 엽니다.

```bash
npm run build   # 프로덕션 빌드
npm run lint    # 린트
```
