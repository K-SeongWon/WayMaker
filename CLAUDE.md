# CLAUDE.md — WayMaker 프로젝트 작업 규칙

이 문서는 이 리포지토리에서 작업하는 모든 AI/사람 협업자가 반드시 지켜야 하는 규칙이다.

---

## ⛔ 0. 최우선 규칙 — 이 리포지토리는 PUBLIC 이다

GitHub `K-SeongWon/WayMaker` 는 **공개 저장소**다. 커밋되는 모든 것은 전 세계에 노출된다.

### 절대 커밋·푸시 금지 대상
- **내부 기획 문서** — `docs/internal/`, `docs/private/` 하위 전체
- **민감 정보** — 실제 교회 이름, 주소, 인적 정보, 장비 시리얼, 내부 IP/네트워크 구성 등
- **`.env`, API 키, 토큰, 비밀번호** 등 모든 시크릿
- **AI 작업 메모/스크래치** — `*.local.md`, `scratch/`, `memory/`
- **사용자가 입력한 실제 배선 데이터** — `*.waymap.local.json`, `sample-data/private/`

이 대상들은 `.gitignore` 로 1차 차단되어 있다. **`.gitignore` 를 신뢰하지 말고**, 커밋 전 항상 `git status` 로 스테이징 목록을 눈으로 확인한다.

### 커밋 전 체크리스트 (매번)
1. `git status` 로 추가될 파일을 **하나씩** 확인한다.
2. `docs/internal/` 또는 민감 파일이 목록에 보이면 **즉시 중단**하고 사용자에게 보고한다.
3. `git add .` / `git add -A` 같은 광범위 스테이징은 **지양**한다. 의도한 파일만 명시적으로 add 한다.
4. 사용자가 명시적으로 요청하기 전에는 **커밋·푸시하지 않는다.**

## 1. 프로젝트 한 줄 요약

교회의 복잡한 A/V·네트워크 장비 결선도와 라우팅 정보를, 클릭·드래그·키보드로 만들고 저장·공유·인쇄하는 웹 도구. 완성 결과물의 명칭은 **WayMap(웨이맵)**.

## 2. 핵심 철학 (모든 설계 판단의 기준)

> **"초보자는 단순하게 필요한 만큼 쉽게 사용. 숙련자는 자신이 원하는 모든 정보를 표시할 수 있도록 지원."**

기능/UI를 추가할 때 항상 자문한다:
- 초보자에게 이 기능이 **부담**이 되는가? → 기본은 숨기고, 필요할 때 펼치는(progressive disclosure) 구조로.
- 숙련자가 표현하고 싶은 정보를 **막고** 있지는 않은가? → 고급 옵션으로 항상 길을 열어둔다.

## 3. 핵심 제약 (사용자 합의 사항)

- **개인정보 무수집.** 어떤 데이터도 서버로 보내지 않는다. 모든 처리는 **로컬(브라우저)** 에서만 이뤄진다.
- **분석/텔레메트리 금지.** 추적 스크립트, 외부 애널리틱스 일절 넣지 않는다.
- **파일 포맷은 범용 JSON** 을 기본으로 한다.
- **라이선스: Apache 2.0** (현재 리포지토리에 포함됨). 출처(NOTICE) 표기 유지.
- **호스팅: Vercel** 등 무료 호스팅 대상. 백엔드 없이 동작 가능한 클라이언트 중심 설계를 우선한다.

## 4. 기획 문서 위치

상세 기획·설계 문서는 **`docs/internal/PROJECT-PLAN.md`** 에 있다 (git 추적 제외). 작업 전 반드시 참고한다.

## 5. 개발 규칙 (스캐폴딩 이후 적용)

- 스택: **Next.js + TypeScript** 중심.
- 사용자 데이터는 항상 로컬(localStorage / IndexedDB / 파일 다운로드)에만 저장.
- 새 의존성 추가 시 라이선스 호환성(Apache 2.0와 충돌 없는지) 확인.
- 커밋 메시지·코드 주석·문서에 실제 교회/개인 정보를 예시로 쓰지 않는다. 예시는 가상의 이름을 쓴다.
